🤖 15+ SDK Agents 📊 LangGraph 🔗 A2A Protocol 📄 OpenAPI 3.0

Overview

The Content Writing Multi-Agent Team is a collaborative AI agent framework that coordinates multiple specialized agents to handle complex content creation tasks. Built on LangGraph for workflow orchestration and A2A Protocol for agent-to-agent communication.

Key Features

🤖 Multiple SDK Support

Built-in support for OpenAI, Salesforce, Microsoft, Google, and 10+ other agent frameworks.

📊 LangGraph Workflows

Graph-based task orchestration with parallel and sequential execution strategies.

🔗 A2A Protocol

Standardized agent-to-agent communication for task delegation and results sharing.

📄 OpenAPI 3.0

REST API specification for easy integration and documentation.

Getting Started

1. Installation

git clone https://github.com/AGenNext/AGenNext-Registry.git
cd AGenNext-Registry
pip install -r requirements.txt

2. Environment Variables

# Create .env file
cp .env.example .env

# Edit with your API keys
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_API_KEY=...
AZURE_OPENAI_KEY=...

3. Quick Start

# View all agent cards
python main.py --mode cards

# Single agent demo
python main.py --mode demo --agent openai --topic "AI Writing"

# Multi-agent team demo
python main.py --mode team --topic "Cloud Computing"

# A2A protocol demo
python main.py --mode a2a

4. Run the Server

# Start A2A server
python main.py --mode server --port 8000

API Reference

Full OpenAPI specification available in openapi.yaml

Base URL

Local:     http://localhost:8000
Production: https://your-domain.com

Endpoints

Health Check

GET /health

Check if the service is running.

curl http://localhost:8000/health

# Response
{"status": "ok", "timestamp": "2026-05-08T01:55:10Z"}

List Agents

GET /agents

Get all available agent cards.

curl http://localhost:8000/agents

# Response
[{"agentId": "openai", "name": "OpenAI Agent", ...}, ...]

Run Agent Task

POST /agents/{agentId}/run

Execute a task with a specific agent.

curl -X POST http://localhost:8000/agents/openai/run \
  -H "Content-Type: application/json" \
  -d '{
    "task": "Write a blog post about AI",
    "content_type": "blog"
  }'

# Response
{
  "agent_id": "openai",
  "content": "# AI Writing...\n\n...",
  "content_type": "blog",
  "quality_score": 0.92
}

Run Team Workflow

POST /team/run

Execute multi-agent team collaboration.

curl -X POST http://localhost:8000/team/run \
  -H "Content-Type: application/json" \
  -d '{
    "task": "Create a complete project",
    "strategy": "parallel"
  }'

# Response
{
  "results": [...],
  "quality_score": 0.95,
  "duration": 12.5
}

A2A Task Operations

POST /a2a/tasks
GET /a2a/tasks/{taskId}

Create and manage A2A protocol tasks.

# Create task
curl -X POST http://localhost:8000/a2a/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "id": "task-001",
    "message": {"role": "user", "parts": [{"type": "text", "text": "Write docs"}]}
  }'

# Check status
curl http://localhost:8000/a2a/tasks/task-001
# {"state": "completed", "result": "..."}

Using Agents Programmatically

from agents import OpenAIAgent, SalesforceAgent, GoogleAgent

# Create agent
agent = OpenAIAgent()

# Run task
result = agent.run(
    task="Write a Python function",
    content_type="code"
)
print(result.content)

Using Custom Providers

from core.llm_client import create_llm_client

# OpenAI
client = create_llm_client("openai", model="gpt-4o")

# Anthropic
client = create_llm_client("anthropic", model="claude-3-opus")

# Local Ollama
client = create_llm_client("ollama", base_url="http://localhost:11434")

# OpenAI-compatible (LM Studio, etc.)
client = create_llm_client(
    "openai-compatible",
    base_url="http://localhost:1234/v1"
)

Adding New Agents

# 1. Create agent class in agents/
# 2. Inherit from BaseAgent
# 3. Register in agents/__init__.py
# 4. Add tests

from agents.base_agent import BaseAgent

class MyAgent(BaseAgent):
    def __init__(self):
        self.agent_id = "my-agent"
        self.name = "My Agent"
        self.capabilities = ["coding", "reasoning"]
